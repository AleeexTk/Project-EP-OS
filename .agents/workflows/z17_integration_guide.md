# 🗺️ EvoPyramid OS: Гайд по Поэтапной Интеграции (Z17-Z1)

Этот документ определяет порядок переноса архитектурных идей из **Визуального Интерфейса (Пирамиды)** в **Физическую Директорию Проекта**.

## 1. Логика Четности (Z-Parity)

Система Z17 — это живой организм, где уровни работают в паре:

- **Нечетные (Z17, Z15, Z13...) — Structural (Кости/Органы)**: Здесь «красуются» важные модули, хранятся манифесты, канонические правила и артефакты. Это видимые узлы силы.
- **Четные (Z16, Z14, Z12...) — Infrastructure (Нервы/Кровь/Черный ход)**: Это «черный ход» для данных. Здесь живут маршрутизаторы, шины, зависимости и сервисы. Они делают всю «грязную работу» по передаче импульса между структурными (нечетными) слоями. Эти слои — не «пустые», они обеспечивают связность.

## 2. Порядок Интеграции (Workflow)

### Шаг А: Проектирование в Интерфейсе

Прежде чем создавать файл, добавь узел в `pyramid_state.json`.

1. Определи **Z-уровень** (Слой Alpha/Beta/Gamma).
2. Геометрия: Центр (9,9). Радиус слоя: **r = 17 - z + 1**.
3. Присвой `kind` (agent/service/memory/protocol).

### Шаг Б: Создание Манифеста

Каждый нечетный узел должен быть зарегистрирован в Каноне:

- Путь: `canon_builder/input/<module_id>/manifest.json`
- Манифест — это "паспорт" модуля, который Пирамида использует для визуализации.

### Шаг В: Физическая Манифестация (Директории)

После валидации в интерфейсе, создается вложенность в соответствии с канонической структурой. Любой новый модуль должен попасть в одну из трех зон:

| Слой | Директория | Z-диапазон | Характер содержимого |
| :--- | :--- | :--- | :--- |
| **Alpha** (\alpha) | `α_Pyramid_Core` | Z17–Z11 | **Канон**: Законы, Память(Z13), Интент, Принципы. |
| **Beta** (\beta) | `β_Pyramid_Functional` | Z9–Z5 | **Runtime**: Агенты (Z9), Движок (Z7), API Gateway (Z7). |
| **Gamma** (\gamma) | `γ_Pyramid_Reflective` | Z3–Z1 | **Рефлексия**: Логи, Мониторинг, Самокоррекция, Пульс. |

### Шаг Г: Реализация Инфраструктуры (Четные уровни)

Для связи между слоями используй **Маршрутизаторы** (Z16, Z14, Z10, Z8, Z4).

- Код инфраструктуры обычно живет в `GLOBAL_NEXUS/` или специальных роутерах.

## 3. Правила Валидации и Приоритеты

1. **Приоритет Рефлексии**: `γ (Reflective) > β (Functional)`. В случае конфликта рантайма с результатами мониторинга — слой Gamma имеет право вето и переписывает решение.
2. **Порог Когерентности**: Автослияние (Auto-Merge) в канон происходит только при `coherence > 0.87`.
3. **Никаких "сирот":** Любой файл в проекте должен принадлежать узлу Пирамиды.
4. **Визуал управляет Диском:** Визуальные изменения в интерфейсе инициируют создание или обновление соответствующих директорий.
