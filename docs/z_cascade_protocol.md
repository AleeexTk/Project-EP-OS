# Каскадная межуровневая проверка со скользящей Z-парой

## Статус

**Canonical Protocol / EvoPyramid Validation Layer**

## Назначение

Каскадная межуровневая проверка со скользящей Z-парой — это нисходящий протокол полной архитектурной валидации EvoPyramid, в котором целостность системы проверяется последовательно на каждой соседней паре Z-уровней.

Протокол применяется для подтверждения того, что смысл, структура, допуски и каноническая согласованность сохраняются при передаче состояния от верхних уровней к нижним.

## Базовый маршрут

Проверка выполняется по скользящей соседней паре:

`Z17–Z16 → Z16–Z15 → Z15–Z14 → ... → Z2–Z1`

Полный цикл охватывает все 17 уровней пирамиды и все 16 межуровневых переходов.

## Принцип

Для каждой Z-пары выполняется один каскадный такт проверки.
Следующая пара может быть активирована только после завершения проверки текущей пары.

Таким образом, ни один нижележащий уровень не считается валидированным, если не подтверждён предыдущий межуровневый переход.

## Трёхтактная структура проверки

### Такт 1. Семантическая целостность

Проверяется, сохраняется ли исходный смысл при переходе между двумя соседними уровнями.

Критерии:

* не потеряно исходное намерение;
* не внесено критическое искажение;
* смысл верхнего уровня корректно проецируется на нижний.

### Такт 2. Межуровневая согласованность

Проверяется, совместимы ли оба уровня как рабочая пара.

Критерии:

* нет конфликта ролей;
* нет канонического противоречия;
* нижний уровень способен принять состояние верхнего без нарушения порядка.

### Такт 3. Кристаллизация перехода

Проверка считается завершённой только после фиксации результата.

Критерии:

* переход зарегистрирован;
* создан след в журнале;
* сформирован статус пары;
* разрешён переход к следующей Z-паре.

## Формальный цикл

Для каждой пары `Zn–Z(n-1)` протокол выполняет:

1. принять состояние верхнего уровня;
2. проверить семантическую целостность;
3. проверить межуровневую согласованность;
4. зафиксировать результат;
5. передать допуск следующей паре.

## Статусы пары

Каждая Z-пара должна иметь один из статусов:

* **PENDING** — пара ожидает проверки
* **ACTIVE** — пара находится в текущем каскадном такте
* **PASSED** — пара успешно валидирована
* **BLOCKED** — переход запрещён
* **DEGRADED** — переход частично допустим, но с отклонениями
* **CRYSTALLIZED** — результат пары зафиксирован и передан вниз

## Условия успешного прохождения

Пара считается пройденной, если:

* семантическая целостность подтверждена;
* межуровневая согласованность подтверждена;
* результат кристаллизован;
* не выявлено критического нарушения канона.

## Условия остановки каскада

Каскад должен быть остановлен, если:

* нарушена целостность смысла;
* обнаружен конфликт между уровнями;
* не выполнена кристаллизация перехода;
* нижний уровень не готов принять состояние верхнего;
* зафиксировано критическое отклонение архитектурного инварианта.

## Роли в протоколе

### Observer

Отслеживает прохождение каскада, фиксирует статус пары и определяет момент остановки или продолжения.

### Monument

Выполняет кристаллизацию результата, закрепляет инварианты и запрещает спуск при нарушении канона.

### Chaos Engine

Собирает сигналы, конфликты и fallback-события, но не утверждает каноническую валидность перехода.

## Артефакты протокола

Каждый каскадный цикл должен оставлять:

* идентификатор пары;
* статус пары;
* след проверки;
* причину блокировки или допуска;
* временную метку;
* ссылку на кристаллизованный результат.

## Каноническое определение

**Каскадная межуровневая проверка со скользящей Z-парой** — это обязательный нисходящий протокол EvoPyramid, обеспечивающий последовательную валидацию соседних Z-уровней от `Z17` до `Z1`, при котором каждый переход проходит проверку смысла, согласованности и кристаллизации до передачи состояния следующей паре.

## Краткая формула

**Понять → согласовать → закрепить → передать вниз.**
